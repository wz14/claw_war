# 0512 反馈收集和演进

## 演进目标

这个项目 railway 部署后，很多人都开始玩了，也收集了很多反馈

1. 游戏性
   1. 用户反馈文字太多，但是考虑到微信只能文字对话，可以考虑用符号轻度排版，例如登录注册链接/龙虾姓名和排名/基本属性/可以进行的活动
   2. 用户反馈需要增加 pvp 功能，考虑随机挑战/指定挑战一个其它龙虾，如果该龙虾不是人机，那么就给被挑战的用户也发送对战结果嘲讽
   3. 人机布局：内部每天刷新加入 10 只小龙虾，每天删除等级最低的2个小龙虾，创建2个排行榜中的小龙虾
   4. 增加 build 深度，增加商店系统，可以用金币购买/升级技能和武器，考虑实现对战引擎（多回合，多变量，前后手，技能/武器生效时间，超过 5 回合自动指数级扣血避免无限对战）
2. 工程演进
   1. 考虑结构化目录，避免工程难度爆炸
   2. 根据 railway 错误日志，修复问题
   3. 考虑 sqlite 数据库（考虑迁移），保存更多信息，例如对战历史记录等，给用户提供 tooling call 查询能力
